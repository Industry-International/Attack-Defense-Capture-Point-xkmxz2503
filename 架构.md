# 攻防战占领点 Mod 架构文档

## 一、项目概述

一个 Minecraft NeoForge 1.21.1 模组，提供攻防战中的据点和区域管理功能。
玩家可以通过**据点方块**、**管理器物品**、**节点图 GUI** 和 **命令** 四种方式管理
据点和区域的占领关系。

## 二、技术栈

| 技术 | 版本 |
|------|------|

| NeoForge | 21.1.226 |
| Java | 21 |
| LDLib2 | 2.2.19 (nodegraphtookit + 自定义UI框架) |
| Gradle | NeoGradle (moddev 2.0.141) |

## 四、核心架构 — 分层数据访问

```
                          ┌─────────────────────────────────┐
                          │       消费者层                    │
                          │  CapturePointBlockEntity         │
                          │  ModCommands / CapturePointGUI   │
                          └──────────┬──────────────────────┘
                                     │ 依赖
                          ┌──────────▼──────────────────────┐
                          │     ICaptureDataAccess 接口      │  ← 统一契约
                          │  (read / write / query / batch)  │
                          └──────────┬──────────────────────┘
                                     │ implements
                          ┌──────────▼──────────────────────┐
                          │   ServerCaptureDataAccess        │  ← 服务端实现
                          │   (单纯委派CaptureManager)       │
                          └──────────┬──────────────────────┘
                                     │ delegates
                          ┌──────────▼──────────────────────┐
                          │   CaptureManager (SavedData)     │  ← 数据持久化层
                          │   points: Map<String, PointEntry> │
                          │   zones : Map<String, ZoneEntry>  │
                          └─────────────────────────────────┘
```

### 设计要点

- **`ICaptureDataAccess`** 定义所有数据操作的统一契约，包含据点 CRUD、区域 CRUD、查询方法、批量快照、版本控制
- **`ServerCaptureDataAccess`** 包装 `CaptureManager`，所有操作单纯委派给底层 SavedData
- 消费者不直接依赖 `CaptureManager`，只依赖接口，便于测试和未来替换实现
- 获取方式：`ICaptureDataAccess.server(serverLevel)`

## 五、数据模型

### CapturePointEntry (据点记录)

```java
record CapturePointEntry(
    String name,          // 唯一名称
    BlockPos pos,         // 世界坐标
    @Nullable String owner, // 所属者 (占领者)
    double radius,        // 判定半径 (1-100)
    int displayColor,     // 显示颜色 (ARGB)
    boolean showRange     // 是否显示范围轮廓
)
```

### ZoneEntry (区域记录)

```java
record ZoneEntry(
    String name,                  // 唯一名称
    List<String> capturePoints,   // 包含的据点名称列表
    @Nullable String requiredZone // 前置依赖区域 (攻防模式)
)
```

### 业务规则

- **区域完全包含据点** — 据点通过 `capturePoints` 列表归属区域
- **区域占领判定** — 区域内所有据点都有 `owner` 时判定为已占领 (100%)
- **攻防模式依赖** — 区域有 `requiredZone` 时，前置区域未占领则此区域不可访问
- **据点和方块的关系** — 据点方块通过 `boundPointName` 绑定到一个据点 (NBT持久化)

## 六、客户端/服务端分离

```
┌──────────────────────────────────────────────────────────┐
│  客户端 (Client)                                         │
│  ┌─────────────────┐  ┌──────────────┐  ┌────────────┐  │
│  │ CapturePointGUI  │  │ BlockEntity  │  │ ManagerItem│  │
│  │ (节点图编辑器)    │  │ (6功能菜单)   │  │ (打开GUI)   │  │
│  └────────┬────────┘  └──────┬───────┘  └────────────┘  │
│           │                  │                            │
│           │  网络包          │  网络包                     │
│           │  (BlockEntity    │  (BlockEntity              │
│           │   ActionPayload) │   ActionPayload)           │
└───────────┼──────────────────┼────────────────────────────┘
            │                  │
┌───────────▼──────────────────▼────────────────────────────┐
│  服务端 (Server)                                           │
│  ┌──────────────────────────────────────────────────┐     │
│  │  BlockEntityActionPayload.handleOnServer()         │     │
│  │  → ICaptureDataAccess.server(level).xxx()          │     │
│  └──────────────────────┬───────────────────────────┘     │
│                         │                                  │
│  ┌──────────────────────▼───────────────────────────┐     │
│  │  CaptureManager (SavedData, 自动持久化到磁盘)      │     │
│  └──────────────────────────────────────────────────┘     │
│                                                            │
│  ModCommands 也通过 ICaptureDataAccess 直接访问 (服务端)   │
└──────────────────────────────────────────────────────────┘
```

### 原则

- **客户端永不直接写 CaptureManager** — 所有写入通过网络包发送到服务端
- **单机模式** — GUI 通过 `getServerCaptureManager()` 获取服务端数据访问（内嵌服务端）
- **专用服务器** — GUI 获取不到服务端数据访问，回退到使用 `/capturepoint` 命令
- **渲染数据** — `CapturePointBlockEntity` 通过 NBT 同步（`syncToClient()` 调用 `level.sendBlockUpdated`）将半径/颜色/显示状态同步到客户端

## 七、网络通信

### BlockEntityActionPayload

自定义网络包 (CustomPacketPayload)，客户端→服务端单向：

```java
record BlockEntityActionPayload(BlockPos blockPos, String action, String data)
```

支持动作：
| action | data | 说明 |
|--------|------|------|
| `create_point_at` | `name,x,y,z,radius` | 创建据点 |
| `set_radius` | `name,radius` | 设置半径 |
| `set_color` | `name,color` | 设置颜色 |
| `toggle_range` | `name,true/false` | 切换范围显示 |
| `add_to_zone` | `zoneName,pointName` | 据点加入区域 |
| `remove_from_zone` | `pointName` | 据点移出区域 |

## 八、节点图系统 (LDLib2 nodegraphtookit)

### 架构

```
CapturePointGraph (extends Graph)
  └─ createGraphModel() → CapturePointGraphModel (extends CustomGraphModelImpl)
       ├─ createWire()    — 端口方向修正 + 兼容性检查
       └─ canAssignTo()   — 信号类型 TypeHandle 严格匹配

CapturePointGraphView (extends GraphView)
  ├─ 隐藏默认面板/标题栏/物品库
  ├─ 右键菜单 → CapturePointGraphDialogs 对话框
  └─ 定时刷新 → CapturePointGraphScreen.refreshNodeTitles()

CapturePointGraphScreen
  ├─ open()        — 创建全屏 UI, 加载数据, 设置回调
  ├─ saveGraph()   — 构建快照 → 冲突检测 → applyGraphSnapshot
  └─ buildSnapshotFromGraph() — 解析节点+连线 → 数据映射
```

### 节点类型

| 节点 | 端口 | 选项 |
|------|------|------|
| **CapturePointNode** (绿色) | 输出: `point_signal` (POINT_SIGNAL) | captured, owner, position, radius, display_color, show_range |
| **CaptureZoneNode** (蓝色) | 输入: `point_in` (POINT_SIGNAL), 输出: `zone_out` (ZONE_SIGNAL), 输入: `required_zone` (ZONE_SIGNAL) | captured, required_zone, points, description, edit_points |

### 连线类型

| 连线 | 含义 | 保存时的处理 |
|------|------|-------------|
| `point_signal` → `point_in` | 据点属于该区域 | 从 WireModel 解析，作为据点列表来源 |
| `zone_out` → `required_zone` | 区域依赖关系 | 从 WireModel 解析，作为 requiredZone 来源 |

### 保存流程

```
用户点击 Save
  ↓
saveGraph()
  ↓
buildSnapshotFromGraph()
  1. 遍历所有 GraphElement → 分类为 PointModel / ZoneModel
  2. 遍历所有 WireModel → 解析连线关系 (wireBasedZonePoints / wireBasedRequiredZone)
  3. 读取 PointModel 选项 → 构建 CapturePointEntry
  4. 读取 ZoneModel 选项 + 连线结果 → 构建 ZoneEntry
  ↓
版本冲突检测 (editMode && snapshotVersion != mgr.getVersion())
  ↓
applyGraphSnapshot(newPoints, newZones) → 原子写入 CaptureManager
```

**关键修复**：区域包含的据点列表**优先从连线(WireModel)解析**，而非仅依赖节点选项 `edit_points`（该选项仅由定时同步回调填充，新建连线后可能尚未更新）。

## 九、UI 系统 (LDLib2 ModularUI)

### GUI 缩放策略

所有对话框宽度使用自适应公式：`Math.min(screenWidth * N%, maxWidth)`

| 组件 | 宽度策略 | 最大宽度 |
|------|----------|----------|
| 方块功能菜单 | `scw * 40%` | 260px |
| 高级配置 | `scw * 55%` | 380px |
| 颜色选择器 | `scw * 50%` | 300px |
| 输入对话框 | 固定 320px | 320px |
| 通知气泡 | `scw * 28%` | 240px |

### 通知系统 (Toast)

QQ 聊天气泡风格，从右侧滑入 → 停留 → 向右滑出。

三种类型：SUCCESS (绿), ERROR (红), INFO (蓝)
由 `ToastLayer.tick()` 驱动动画，`GraphView.screenTick()` 中每帧调用。

## 十、方块系统

### CapturePointBlock

- 继承 `BaseEntityBlock`，渲染模式 `RenderShape.MODEL`
- 右键触发 `CapturePointBlockEntity.openUI()`（客户端打开菜单）

### CapturePointBlockEntity

- 6 个功能按钮：创建据点、绑定区域、查看状态、设定半径、切换范围、移除绑定
- 本地缓存：`boundPointName`, `radius`, `displayColor`, `showRange`
- NBT 持久化：这些字段全部保存到 `saveAdditional/loadAdditional`
- 所有写入操作通过 `sendAction()` → `BlockEntityActionPayload` 网络包通信

## 十一、命令系统

### /capturepoint 命令树

需要权限等级 2 (op)：

```
/capturepoint
├── help                               — 显示帮助
├── gui                                — 打开GUI提示
├── create <name>                      — 在玩家位置创建据点
├── remove <name>                      — 移除据点
├── setowner <name> <owner>            — 设置所属
├── clearowner <name>                  — 清除所属
├── list                               — 列出所有据点和区域
├── createat <name> <x> <y> <z> [r]   — 在坐标创建据点
├── setradius <name> <radius>          — 设置半径
├── setcolor <name> <color>            — 设置颜色
├── settoggle <name> <true/false>      — 切换范围显示
├── removefromallzones <name>          — 移出所有区域
├── relationships                      — 查看关系
├── savegraph                          — 保存 (占位, GUI操作)
└── zone
    ├── create <name> [requires]       — 创建区域
    ├── remove <name>                  — 移除区域
    ├── addpoint <zone> <point>        — 添加据点
    ├── removepoint <zone> <point>     — 移除据点
    ├── setrequired <zone> [req|clear] — 设置依赖
    └── status                         — 区域状态
```

所有命令通过 `ICaptureDataAccess.server(level)` 访问数据，不直接操作 CaptureManager。

## 十二、渲染系统

### CapturePointBlockEntityRenderer

- `showRange=true` 时渲染圆形轮廓
- 32 段圆环 (水平) + 4 条垂直辅助标记
- 颜色来自 `displayColor` (ARGB)
- 渲染距离 128 格
- 使用 `RenderType.LINES` 绘制

## 十三、国际化 (i18n)

所有文本使用 `Component.translatable(key)`，包含两个语言文件：

- `assets/attack_defense_capture_point_xkmxz/lang/en_us.json` — 英文
- `assets/attack_defense_capture_point_xkmxz/lang/zh_cn.json` — 中文

覆盖范围：命令反馈、GUI 文本、节点名称、端口名称、选项标签、Toast 消息、对话框文本。
